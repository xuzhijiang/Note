# 当MySQL单表记录数过大时的优化方法(大表优化)

    当MySQL单表记录数过大时，数据库的CRUD性能会明显下降，一些常见的优化措施如下：
  
    1. 限定数据的范围： 比如：当用户在查询订单历史的时候，我们可以控制在一个月的范围内
    2. 读/写分离：* 主库负责写，从库负责读
    3. 水平和垂直分区
